# /explain-concept

對任何概念進行八維解剖，壓縮為一個頓悟。

靈感來源：[lijigang/ljg-explain-concept](https://github.com/lijigang/ljg-explain-concept)

## 觸發條件
- 使用者說 `/explain-concept {概念}` 或「幫我拆解 XX 這個概念」
- 使用者想要深度理解某個概念的本質

## 不要觸發
- 使用者只是問一個快速問題（如「什麼是 ZK proof」） → 直接簡短回答
- 使用者問技術操作（如「怎麼用 Git」） → tech-tutor agent
- 使用者想討論某篇文章的觀點 → 直接對話或 `/digest`
- 概念太狹窄或太具體（如「React useState」） → 直接解釋就好

## 執行步驟

### 1. 預處理
1. **澄清定義**：最通行的定義是什麼？常見誤解有哪些？
2. **識別核心詞素**：概念中隱含的關鍵構成元素

### 2. 八維探索
依次從 8 個維度深度探索。每個維度產出 2-3 句**精煉洞見**，不要水詞。

1. **歷史溯源**：最早出處 → 演變脈絡 → 當代含義的轉折點
2. **辯證分析**：提出反題（對立面是什麼？）→ 尋找綜合 → 否定之否定後的更高理解
3. **現象學還原**：懸置所有預設，回到「事物本身」→ 本質直觀 → 用一個生活場景還原
4. **語言學解構**：詞源拆解（中/英/希臘/拉丁）→ 語義場（相鄰概念網絡）→ 符號學隱喻
5. **數學形式化**：嘗試寫出一個公式或形式化表達 → 公式的邊界條件
6. **存在主義審視**：這個概念對「人如何存在」意味著什麼？
7. **美學維度**：概念有沒有「美」的一面？審美特質是什麼？用一個具體意象呈現
8. **元哲學反思**：我們用什麼隱喻在理解它？這個隱喻遮蔽了什麼？換一個隱喻會怎樣？

### 3. 內觀與提煉
1. **從概念內部看世界**：如果你「是」這個概念本身，你看到的世界是什麼樣的？（第一人稱視角，3-5 句）
2. **提煉核心**：八維探索中，哪些洞見指向同一個深層結構？提取出來。

### 4. 頓悟壓縮
1. **The One 公式**：`概念 = ...`（一個公式）
2. **一句話**：費曼式表達——用最簡單的語言說出最深的理解
3. **拓撲圖**：用純 ASCII 字元畫出概念的結構關係

### 5. Vault 連結
搜尋 Vault 中與該概念相關的筆記，在產出中加入 wikilink 連結。

### 6. 生成與寫入
先用 `date` 確認日期。

## Skill 網絡
- 概念來源：`/digest`（外部文章中的概念）、`/twitter-capture`（推文中的概念）

- 相關 MOC：[[moc-技術]]、[[moc-learning]]

## 重要提醒
- 每個維度要**精煉**，不要為了湊篇幅而灌水
- 充分運用模型知識，不限制在 Vault 內容
- 這是分析工具，不是使用者自己的思考——產出放 `03_Output/` 而非 `02_Thinking/`
- 如果使用者想基於這個解剖寫自己的思考，引導他到 `02_Thinking/`
