# /digest

為 Inbox 項目建立結構化摘要，提取論點、找出爭議、連結既有知識。

## 觸發條件
- 使用者說「digest [檔名]」「幫我做摘要」
- 使用者指定某個 Inbox 檔案要處理

## 不要觸發
- 使用者只是想**聊**某篇文章的觀點 → 直接對話討論
- 使用者想自己寫想法 → 引導到 `02_Thinking/`
- 使用者問某個概念是什麼 → `/explain-concept` 或直接回答
- 來源不在 `00_Inbox/` → 先確認是否要建 Inbox 檔案
- 使用者說「看看這個」但沒明確說要做摘要 → 先問意圖
- 使用者想把多篇整合成一個框架 → 用 knowledge-synthesizer agent

## 執行步驟

### 1. 讀取來源
從 `00_Inbox/` 讀取指定檔案。

### 2. 搜尋相關筆記
搜尋 Vault 中與該主題相關的既有筆記（用 Grep/Glob 或語意搜尋工具），準備做 wikilink 連結。

### 3. 分析內容
- 提取核心論點
- 標記論證結構
- 找出爭議點或值得深思的地方

### 4. 建立 Digest
使用下方模板，填入分析結果。特別注意：
- 「與我現有認知的關聯」要用具體 wikilink，不是泛泛的「相關」
- 對使用者現有系統做審計（已做到/沒做/不需要）

### 5. 判斷引導
Digest 完成後，問使用者：

```
💭 這篇內容可能導致什麼結果？
（有感覺就寫一句，沒有就跳過）
```

如果使用者有判斷，填入 `[!prediction]` 區塊：
- **預測**：使用者的原話，保持簡短
- **影響標的/領域**：具體的標的或產業
- **時間範圍**：多久後可以驗證（例如「3 個月」「年底前」）
- **信心度**：1-10
- **回顧日期**：根據時間範圍自動計算，格式 YYYY-MM-DD

如果使用者跳過，保留空白模板（含註解「沒有判斷就留空」）。

### 6. 儲存
儲存到 `01_Digest/`，命名規則：`digest-{分類}-{標題關鍵字}-{作者}.md`

## 檔案位置
- 來源：`00_Inbox/`

## 重要提醒
Digest 是**分析外部內容**，不是使用者自己的思考。
使用者自己的思考應該放在 `02_Thinking/`。

## Skill 網絡
- 上游：`/twitter-capture`（擷取內容 → Inbox）
- 並行：`/inbox-status`（查看待處理項目）
- 分流：白皮書/技術文件 → `/whitepaper`（不走 digest）
- 整合：knowledge-synthesizer agent（多篇合併成框架）
- 拆解：`/explain-concept`（單一概念深度解剖）
- 下游：`/prediction-review`（掃描 digest 裡的 `[!prediction]`，月度標記結果，季度產出回顧報告）
- 輸出至：[[moc-技術]]、[[moc-investing]]、[[moc-learning]]（依主題更新 MOC）

每次 digest 被使用者糾正或稱讚時，更新這個區塊。
執行 `/digest` 時**必須**先讀這些規則。
