# /handoff

在長對話結束前，建立結構化的交接文件，讓下一個 session 能無縫接續。

## 觸發條件
- 使用者說「handoff」「交接」「幫我記錄進度」
- 對話即將結束、context 快滿時

## 不要觸發
- 對話剛開始或很短 → 沒有什麼需要交接的
- 使用者只是在閒聊 → 不需要正式交接
- 使用者想做週報 → `/weekly-review`

## 為什麼需要這個
Claude Code 的對話有 context window 限制。長 session 中做了大量工作後，如果直接開新對話，所有進度和脈絡都會遺失。HANDOFF 文件是「寫給下一個自己的備忘錄」。

## 執行步驟

### 1. 盤點現狀
回顧整個對話，整理：
- 完成了什麼
- 正在進行什麼
- 計畫但還沒開始的
- 遇到的問題或決策點

### 2. 清理舊 handoff
- 刪除 `.claude/handoffs/` 中超過 7 天的檔案（用檔名中的日期判斷）
- 刪除前，檢查每份 handoff 是否有「進行中」項目還沒出現在 `_workflow/now.md`
- 如果有，先更新 NOW.md 再刪除

### 3. 建立 HANDOFF 文件
儲存到 `.claude/handoffs/handoff-{YYYY-MM-DD}-{主題}.md`

### 3.5 更新 NOW.md
用以下邏輯覆寫 `_workflow/now.md`：
- **當前焦點**：從「進行中」挑最重要的 1-2 項
- **進行中**：所有「進行中」項目的一行摘要
- **等待中**：所有「待開始」但有前置條件的項目
- **最近決策**：本次 session 的決策紀錄（最多 3 條）
- **提醒**：注意事項中跨 session 有效的部分
- 更新 frontmatter `last_updated` 和 `updated_by: handoff`

### 4. 更新 Memory（如適用）
如果 session 中發現了值得跨 session 記住的模式或偏好，同時更新 `MEMORY.md`。

## 模板

````markdown
---
type: handoff
date: {YYYY-MM-DD}
session-topic: {一句話描述這次 session 的主題}
---

# HANDOFF: {主題}

## Session 摘要
{2-3 句描述這次 session 做了什麼}

## 已完成
- [ ] {完成項目 1}
- [ ] {完成項目 2}

## 進行中（需要繼續）
- {項目描述}
  - 目前狀態：{...}
  - 下一步：{...}
  - 相關檔案：{wikilinks}

## 待開始
- {項目描述}
  - 背景：{為什麼要做}
  - 前置條件：{需要先完成什麼}

## 決策紀錄
{這次 session 中做的重要決策，以及為什麼}

## 注意事項
{下一個 session 需要知道的 gotchas、限制、或特殊情況}

## 相關檔案
{列出這次 session 建立或修改的所有檔案路徑}
````

## 原則
- **寫給不知道任何背景的 Claude** — 不要假設下一個 session 知道這次做了什麼
- **具體勝過抽象** — 寫檔案路徑，不要寫「那個檔案」
- **決策理由比決策本身重要** — 記下「為什麼」，不只是「做了什麼」
- **行動導向** — 每個「待做」項目都要有明確的下一步

## 使用情境範例
- 處理了 30 篇 Inbox，還有 12 篇待處理 → 記錄哪些完成、用了什麼分類邏輯
- 建某功能，改了 5 個檔案 → 記錄架構決策、測試狀態
- 研究了 3 個投資標的 → 記錄研究結論、開放問題
