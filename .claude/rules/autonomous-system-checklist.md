# 自主系統部署檢查

## 觸發條件

使用者要求部署、安裝、設定**會持續消耗按量計費資源**的自主系統時，自動執行此檢查。

例如：
- AI agent（自動化代理等）
- 會呼叫付費 API 的背景服務或排程任務
- 自動交易機器人
- 雲端服務（AWS/GCP 持續計費的資源）

**不觸發**：純本地、固定成本的排程任務（備份、清 log、本地腳本）。

## 三件套檢查

部署前必須確認以下三項都已設定，缺一不可：

### 1. 天花板（Ceiling）— 硬上限，超過就停
- API daily/monthly spend limit
- Token per request 上限（防歷史膨脹）
- Session 自動清理或 rotation

### 2. 儀表板（Meter）— 隨時能看到狀態
- 目前花費（今天/本週/本月）
- 每次 request 的量級趨勢（是否在膨脹）
- 系統是否正常運行

### 3. 警報器（Alarm）— 接近上限時主動通知
- 花費到 50% / 80% 時通知
- 單次 request 異常大時通知
- 系統錯誤時通知

## 行為

1. 部署完功能後、啟動前，逐項跟使用者確認三件套
2. 如果某項還沒設定，主動提出實作方案
3. 三項都確認後才啟動系統
4. 如果使用者說「先跑再說」，提醒風險但尊重決定，並在對話結尾再提一次
